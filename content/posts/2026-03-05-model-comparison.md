# OpenClaw 模型选择指导手册（国内外主流模型）

## 1. 背景：被 Claude Max 额度卡脖子的反思

这周三晚上，我正准备让龙虾哥跑个批量任务，Claude API 却直接甩给我一句："You're out of extra usage · resets Mar 6 at 11am (Asia/Shanghai)"——Max 额度用完了，离周五重置还剩两天。

![Claude 额度耗尽提示](resources/images/claude-usage-limit.jpg)

回头看我的 OpenClaw 配置：主会话用 Opus 4.6，子 agent 默认 Sonnet 4.6；遇到我觉得“活很小”的时候，也会手动指定 OpenClaw 用 Gemini 或 Kimi。一个月下来烧了不少钱。冷静想想，其实很多任务根本不需要这么贵的模型：

- 抓个网页、格式化个 markdown，用 Sonnet 是杀鸡用牛刀
- 写个简单脚本、跑个 cron 任务，Sonnet 也有点浪费
- 前端代码审查、批量处理，完全可以用更便宜更快的模型

于是我花了几个小时时间，系统调研了截至 2026 年 3 月的国内外主流 AI 模型，目标很明确：**为 OpenClaw 的各种子任务找到性价比最优的模型组合，并准备好动态路由与 Fallback**。

这篇文章就是调研成果的总结，既是给自己的备忘录，也分享给同样在用 OpenClaw 或其他 AI Agent 框架的朋友们。

## 2. 关键结论（30秒版）

先把结论放前面，方便你直接拿去给 OpenClaw 做子任务路由：

- **日常默认**：海外优先 Claude **Sonnet 4.6**；国内可用优先 **MiniMax M2.5**
- **轻量任务**（抓取/总结/翻译/格式化/批量脚本）：海外优先 Gemini **3 Flash** 或 **gemini-3.1-flash-lite**；国内可用优先 **豆包 Seed 2.0 Mini**
- **超长上下文**（超长文档/大代码库通读、跨文件串联）：国内优先 **Kimi K2.5**
- **复杂软件工程闭环**（多文件重构 + 终端调试 + 长时自治）：海外优先 OpenAI **GPT-5.3-Codex**；国内可用先用 **MiniMax M2.5** 顶上（更适合终端密集与快速迭代）
- **复杂推理/高价值决策**：海外优先 Gemini **3.1 Pro**；国内可用优先 **Qwen3-Max-Thinking**
- **中文注释/业务老项目**：海外可用优先 Claude **Sonnet 4.6**；国内优先 Qwen **3.5-Plus**
- **工具调用/界面生成/PPT**：海外可用优先 Claude **Sonnet 4.6**；国内优先 **GLM-4.7**

## 3. 模型对比：海外巨头 vs 国内主力

这次调研的信息全部来自公开资料：官方博客、API 文档、第三方测评文章，以及社区反馈。

**特别说明：对于本文涉及的所有模型（无论是海外还是国内），以下评价均主要基于公开的测试数据和社区开发者的真实反馈提取，并非深度实战结论。**

### 3.1 海外三巨头（Claude / Gemini / Codex）+ 实时信息补位（Grok）

#### Claude 4.6（Anthropic）

- **Sonnet 4.6**：SWE-Bench Verified 约 79.6%，速度快，成本 $3/$15（入/出，每百万 token）
- **Opus 4.6**：旗舰版，SWE-Bench Verified 约 80.8%，GPQA Diamond 约 91.3%，成本更高（公开资料常见口径为 $5/$25）

**社区反馈**：Sonnet 在日常编码、代码审查上性价比极高。Opus 和 Sonnet 在编程基准上差距只有 1.2%，但在复杂推理（GPQA）上 Opus 领先明显。对于非极端复杂的场景，推荐优先使用 Sonnet 以控制成本。

#### Gemini 3.1（Google）

Gemini 3.1 是 2026 年 2 月的最新版本，分 Pro 和 Flash 两款：

- **Flash**：速度极快（Flash Lite 约 363 tok/s；Flash 以官方口径为准），成本约 Pro 的 1/4，SWE-Bench Verified 约 78%（同口径以官方更新为准），适合批量处理、实时应用。**⚠️ 注意：Gemini 3.1 Flash 标准版 API 尚未正式发布（截至 2026-03-05），目前可用的是旧版 gemini-3-flash 或预览版 gemini-3.1-flash-lite。**
- **Pro**：旗舰推理模型，GPQA Diamond **94.3%**（13/16 基准第一），SWE-Bench Verified **80.6%**（与 Claude Opus 4.6 的 80.8% 接近）

**社区反馈**：Flash 是日常代码审查、抓取总结、格式化翻译的首选，速度快成本低；Pro 适合复杂推理、科学数学、企业级 Agent。**Gemini 3.1 Pro 在推理基准（如 GPQA）上领先 Opus，但写作质量上 Claude 仍有优势。**

#### OpenAI GPT-5.3-Codex

Codex 是 OpenAI 专门为编程 Agent 打造的模型，2026 年 2 月发布 5.3 版本：

- **SWE-Bench Pro**：约 56.8%
- **Terminal-Bench 2.0**：约 77.3%（长周期自治能力）
- **上下文**：400K（max output 128K）
- **特色**：支持全流程开发（编码→调试→部署→监控→文档），可以充当「AI 同事」协作数小时

**适用场景**：复杂多文件重构、全流程软件工程、自主 Web/游戏开发。**Codex 的优势在于长时自治和全流程能力，但日常编码性价比不如 Claude Sonnet 或 Gemini Flash。**

#### Grok 4.2（xAI）

Grok 的定位更偏“**实时信息 + 搜索增强**”，适合把最新新闻、社交讨论、热点事件这类信息快速拉齐，再做判断与总结。

- **适用场景**：新闻/社交热点跟进、需要强实时性的事实核查、对“刚发生的变化”敏感的研究与分析
- **不太适合**：纯编程/软件工程闭环（这类任务更适合 Sonnet / Codex / Flash），以及对成本极度敏感的高并发批处理

### 3.2 国内主力：千问、豆包、DeepSeek、MiniMax、Kimi、GLM

#### 千问 Qwen3.5-Plus（阿里）

- **版本**：Qwen3.5-Plus（2026-02-16 发布，当前旗舰）
- **参数**：397B MoE，17B 激活
- **上下文**：100 万 token
- **特色**：多模态（文本/图像/视频），支持 201 种语言，中文代码/注释/规范支持最好
- **SWE-Bench**：开源版 76.4%
- **社区反馈**：**Qwen 3.5 在中文代码、本土规范、遗留代码理解上表现优秀，社区反馈极强**

**适用场景**：中文代码库、本土规范、遗留代码理解、私有化部署。

#### 豆包 Doubao Seed 2.0（字节跳动/火山引擎）

- **版本**：Seed 2.0（2026-02-14 发布），含 Pro / Lite / Mini / Code 四款
- **SWE-Bench Verified**：Pro/Code 76.5%，Lite 73.5%，Mini 67.9%
- **特色**：支持图像输入（设计稿/截图→代码），与 TRAE 深度集成，兼容 Anthropic API
- **成本**：Pro 约 3.2 元/百万 token 入，比海外同级低约一个数量级

**社区反馈**：豆包在前端视觉转代码（截图出代码）、企业官网等相对确定的场景上表现不错。但在复杂的后端核心逻辑开发中，虽然基准分很高，开发者反馈仍建议先小规模试水，**毕竟后端开发质量和架构稳定性才是命脉，成本只是次要考量。**

#### DeepSeek V3.2 / R2（深度求索）

- **版本**：V3.2（2025-12 正式版），R2（推理专用）
- **参数**：671B MoE，激活 37B，128K 上下文
- **编程基准**：ICPC/IOI 竞赛金牌级，代码能力接近 Claude Sonnet
- **推理基准**：R2 在 AIME 2024 约 92.5%，MATH-500 约 99.1%
- **成本**：API 极低，约海外同级的 1/5–1/10

**社区反馈**：DeepSeek 在数学/算法/竞赛级编程上表现优秀，成本极低，适合成本敏感的推理与代码任务。

#### MiniMax M2.5

- **参数**：230B MoE，约 10B 激活，204K 上下文
- **SWE-Bench Verified**：**80.2%**（超 Claude Opus 4.6）
- **速度**：官方口径约 50 tok/s；Lightning 档约 100 tok/s（随并发与上下文波动）
- **成本**：约省 80%+（vs Kimi），约 ¥29/月起

**社区反馈**：MiniMax 在快速迭代、实时交互、Shell/终端密集任务上表现优秀，速度快成本低。

#### Kimi K2.5（月之暗面）

- **参数**：1T 总参数，32B 激活，MoE（384 专家），256K 上下文
- **SWE-Bench Verified**：76.8%
- **特色**：原生视觉（图像/视频）、Video to Code、Agent Swarm（100 子 Agent、1500 次工具调用）
- **成本**：约 34 tok/s，部分场景较 GPT-5.2 省 5–21 倍

**社区反馈**：Kimi 在超长文档、数学/科学/法律推理、Agent 集群与复杂工具链上表现优秀。

#### GLM-4.7（智谱 AI）

- **版本**：GLM-4.7（2025-12-23 发布）
- **SWE-Bench Verified**：73.8%（开源第一，超 Claude Sonnet 4.5）
- **特色**：三层思考机制，界面生成（PPT 16:9 适配率 52%→91%），Artifacts 实时预览
- **成本**：约 Claude 的 1/7

**社区反馈**：GLM 在全栈/网页/PPT 生成、工具调用密集任务上表现不错，成本优势明显。

## 4. 名词解释：看懂测试基准

调研过程中遇到了很多专业测试名词，这里用通俗语言解释一下：

### SWE-Bench Verified

**全称**：Software Engineering Benchmark Verified  
**测什么**：真实 GitHub Issue 修复能力  
**怎么测**：给 AI 一个真实的 GitHub Issue（比如"修复这个 Bug"），看它能不能自己写代码、跑测试、提交 PR  
**分数含义**：79.6% 表示 100 个 Issue 里能成功修复约 80 个  
**为什么重要**：这是最接近真实编程工作的测试，比单纯的代码补全更能反映 AI 的实战能力

### GPQA Diamond

**全称**：Graduate-level Professional Question Answering  
**测什么**：研究生级别的专业问题推理能力  
**怎么测**：给 AI 一些物理、化学、生物、数学等领域的研究生考试题，看它能答对多少  
**分数含义**：94.3% 表示答对了 94.3% 的题目  
**为什么重要**：测试 AI 的深度推理能力，不是简单的知识记忆，而是需要多步逻辑推理

### LiveCodeBench

**全称**：Live Code Benchmark  
**测什么**：实时代码生成能力  
**怎么测**：给 AI 一个编程题目（类似 LeetCode），看它能不能写出正确的代码并通过测试用例  
**分数含义**：87.8 表示在 100 道题里能正确解决约 88 道  
**为什么重要**：测试 AI 的算法能力和代码正确性

### Terminal-Bench

**全称**：Terminal Benchmark  
**测什么**：命令行/终端操作能力  
**怎么测**：给 AI 一个任务（比如"部署一个 Web 应用"），看它能不能自己执行 Shell 命令、调试错误、完成任务  
**分数含义**：77.3% 表示能成功完成约 77% 的终端任务  
**为什么重要**：测试 AI 的自主操作能力，不只是写代码，还要能跑起来

### τ²-Bench（Tau-Bench）

**全称**：Tool and Agent Use Benchmark  
**测什么**：工具调用和 Agent 协作能力  
**怎么测**：给 AI 一个复杂任务（比如"生成一个 PPT"），看它能不能正确调用工具（API、命令行工具等）完成任务  
**分数含义**：87.4 表示能正确完成约 87% 的工具调用任务  
**为什么重要**：测试 AI 在 Agent 框架中的实用性，OpenClaw 这类框架就需要这个能力

### AIME

**全称**：American Invitational Mathematics Examination  
**测什么**：美国数学邀请赛题目  
**怎么测**：给 AI 一些高中数学竞赛题，看它能答对多少  
**分数含义**：96.1% 表示答对了 96.1% 的题目  
**为什么重要**：测试 AI 的数学推理能力，DeepSeek R2 和 Kimi K2.5 在这个测试上表现优秀

### ARC-AGI

**全称**：Abstraction and Reasoning Corpus for Artificial General Intelligence  
**测什么**：抽象推理能力（接近 AGI 的测试）  
**怎么测**：给 AI 一些视觉模式识别题，看它能不能找到规律并推理出答案  
**分数含义**：77.1% 表示能正确推理约 77% 的题目  
**为什么重要**：这是最接近「通用智能」的测试，Gemini 3.1 Pro 在这个测试上较前代翻倍

## 5. 总结

这次调研让我对 AI 模型的选择有了更清晰的认识：

1. **不是所有任务都需要 Opus**：85% 的日常任务用 Sonnet 或 Flash 就够了，Opus 留给真正复杂的场景
2. **国内模型值得关注**：千问、豆包、DeepSeek、MiniMax 在编程和推理基准上已经接近甚至超越海外模型，成本优势明显
3. **速度和成本同样重要**：Gemini Flash 和 MiniMax M2.5 的速度优势在批量任务和实时交互中非常明显
4. **专用模型更高效**：Codex 适合全流程开发，豆包 Code 适合视觉转代码，Kimi 适合 Agent 集群，选对模型事半功倍
